# TrendRadar 金融跟踪系统 - 完整设计方案

## 概览

TrendRadar 金融跟踪系统是一个**渐进式增强**的智能投资辅助系统，通过分析热点新闻与金融市场的关联，为用户提供数据驱动的投资洞察。

### 核心理念

**从数据采集到智能决策的完整闭环**

```
Phase 1         Phase 2           Phase 3          Phase 4
数据基础   →   关联分析   →   异常检测   →   投资建议
  ↓              ↓              ↓              ↓
行情跟踪      统计模式       风险预警       决策支持
```

### Phase 2: AI 关联分析

**文档**: [phase2-correlation-analysis.md](phase2-correlation-analysis.md)

**功能**：
- 记录"热点出现 → 价格反应"事件
- AI 情绪分析（积极/消极/中性）
- 统计关联模式（胜率、平均涨跌幅）
- 生成关联洞察（"新能源热点出现时，宁德时代有 65% 概率上涨"）

**核心模块**：
- `CorrelationAnalyzer`：关联分析器
- 数据库表：
  - `hotspot_price_events`：热点-价格事件记录
  - `correlation_patterns`：统计模式

**关键指标**：
- **胜率**：热点出现时标的上涨的概率
- **平均涨跌幅**：上涨/下跌的平均幅度
- **情绪分数**：AI 分析的新闻情绪（-1 到 1）

**数据积累策略**：
- 前 5-7 天：**静默积累**（不显示给用户）
- 7 天后：开始展示简单洞察
- 30 天后：完整功能上线

**推送示例**：
```
🧠 智能洞察
「新能源」热点历史关联：
• 宁德时代 (300750)
  - 胜率: 68%（17胜 8负）
  - 平均涨幅: +3.2%（上涨时）
  - 情绪: 积极（0.65）
  
💡 提示: 该关联已验证 25 次
```

**前置条件**: Phase 1 运行至少 5-7 天

**实施周期**: 3-4 周

### Phase 3: 价格异常检测

**文档**: [phase3-anomaly-detection.md](phase3-anomaly-detection.md)

**功能**：
- 统计基线计算（均值、标准差、波动率）
- 多维度异常检测（价格、成交量、趋势）
- 分级预警（高/中/低）
- 智能推送（去重、频控）

**核心模块**：
- `StatisticsCalculator`：统计基线计算器
- `AnomalyChecker`：异常检查器
- `AnomalyDetector`：主检测器
- 数据库表：
  - `price_statistics`：统计基线
  - `anomaly_events`：异常事件记录
  - `user_watchlist`：用户关注列表

**检测维度**：
1. **价格异常**：Z-score > 2.5σ（突然大涨/大跌）
2. **成交量异常**：成交量 > 均值 2 倍（放量）
3. **连续异常**：连续 3 日同向波动 > 5%（趋势确认）

**异常分类**：
- `surge`：突然大涨
- `crash`：突然大跌
- `volume_spike`：成交量异常
- `consecutive`：连续异常

**严重程度**：
- `high`：Z-score > 3.0 且成交量 > 3 倍
- `medium`：Z-score > 2.5 且成交量 > 2 倍
- `low`：其他异常情况

**推送示例**：
```
🚀 【严重异常】价格异常提醒

宁德时代 (300750)
类型: 突然大涨
当前价格: 268.50
涨跌幅: +8.5%
成交放大: 3.2倍

📊 统计对比:
  正常涨跌幅: +0.5% ± 2.1%
  偏离程度: 3.8σ

关联热点: 新能源, 电池技术

💡 建议: 异常放量大涨，建议关注是否有重大利好消息
```

**独立性**：
- ✅ 可独立于 Phase 2 运行（仅基于统计）
- ✨ 配合 Phase 2 效果更佳（结合情绪数据）

**前置条件**: Phase 1 运行至少 30 天（统计基线需要充足样本）

**实施周期**: 4-5 周

### Phase 4: 投资建议

**文档**: [phase4-investment-advisor.md](phase4-investment-advisor.md)

**功能**：
- 多维度信号聚合（关联 + 情绪 + 异常）
- 风险收益量化评估
- 概率化投资建议（不承诺收益）
- 投资组合优化
- 建议表现追踪

**核心模块**：
- `DataAggregator`：数据聚合器
- `SignalEvaluator`：信号评估器
- `RiskCalculator`：风险计算器
- `RecommendationGenerator`：建议生成器
- `InvestmentAdvisor`：主入口
- 数据库表：
  - `investment_recommendations`：投资建议
  - `recommendation_tracking`：建议跟踪
  - `strategy_performance`：策略表现

**决策逻辑**：
```python
综合置信度 = 
    关联强度 * 50% +
    情绪分数 * 20% +
    异常信号 * 20% +
    样本充分性 * 10%

if 置信度 >= 70% and 胜率 >= 60%:
    action = 'buy'
elif 置信度 >= 50% and 胜率 >= 55%:
    action = 'watch'
else:
    action = 'hold' or 'avoid'
```

**风险评估**：
- **预期收益**：历史平均涨跌幅
- **盈利概率**：历史胜率
- **最大回撤**：历史最差表现
- **止损建议**：基于波动率的 2σ

**推送示例**：
```
💼 智能投资建议

✅ 建议买入
宁德时代 (300750)
置信度: 72%

📊 预期表现:
  期望收益: +3.8%
  盈利概率: 68%
  最大回撤: -4.2%

💡 建议:
  止损价: 236.50
  持仓周期: 7-14d
  有效期至: 2026-05-03

📝 理由:
  关联性: 强相关（30次历史事件）
  情绪: 积极
  异常信号: 上涨信号
  历史最好: +12.3%
  历史最差: -6.8%

⚠️ 风险提示:
  风险可控，但请注意市场变化

━━━━━━━━━━━━━━━━━━
⚠️ 免责声明:
本建议基于历史数据统计，不构成投资建议。
投资有风险，入市需谨慎。
```

**设计原则**：
1. **概率思维**：不承诺收益，只提供概率
2. **数据充分性**：最少 20 次历史事件
3. **风险优先**：所有建议包含风险评估
4. **多维度评分**：综合多个信号源
5. **组合思维**：推荐组合而非单一标的

**前置条件**: 
- **必需**: Phase 1 + Phase 2 运行至少 30 天
- **可选**: Phase 3（增强效果）

**实施周期**: 6-8 周（包括回测和法律审查）

**重要限制**：
- ⚠️ **法律风险**：需确认投顾资质要求
- ⚠️ **免责声明**：必须明确告知风险
- ⚠️ **数据依赖**：质量完全依赖历史数据
- ⚠️ **黑天鹅事件**：无法预测突发事件

## 配置管理

### 配置文件结构

**主配置**: `config/config.yaml`

```yaml
finance:
  enabled: true
  mapping_file: "finance_mapping.yaml"
  
  # Phase 1: 基础配置
  display:
    max_items_per_keyword: 3
    show_in_push: true
    show_volume: true
  
  data_fetch:
    timeout: 10
    retry_times: 2
    cache_ttl: 300
  
  # Phase 2: 关联分析
  correlation_analysis:
    enabled: true                      # 开关
    min_events_to_show: 5              # 最少事件数（开始展示）
    silent_accumulation_days: 5        # 静默积累天数
    ai_sentiment:
      enabled: true                    # AI 情绪分析开关
      provider: "openai"               # 复用现有配置
  
  # Phase 3: 异常检测
  anomaly_detection:
    enabled: true
    baseline_window: 30
    thresholds:
      price_z_score: 2.5
      volume_ratio: 2.0
      consecutive_days: 3
    notification:
      enabled: true
      min_severity: 'medium'
      cooldown_hours: 24
  
  # Phase 4: 投资建议
  investment_advisor:
    enabled: false                     # 默认关闭（需用户主动开启）
    min_samples: 20
    min_confidence: 0.4
    notification:
      enabled: true
      min_confidence_for_push: 0.6
      actions_to_push: ['buy', 'watch']
    disclaimer:
      enabled: true
```

## 常见问题（FAQ）

### Q1: 为什么要分四个阶段？

**A**: 渐进式设计有三大优势：
1. **降低风险**：每个阶段独立验证，避免一次性大改
2. **数据驱动**：后续阶段依赖前期数据积累，必须分步
3. **灵活调整**：根据实际效果决定是否继续后续阶段

### Q2: Phase 2-4 可以跳过某个阶段吗？

**A**: 依赖关系：
- Phase 3 **可以跳过 Phase 2**（独立运行）
- Phase 4 **必须依赖 Phase 2**（需要关联数据）
- 建议按顺序实施，效果最佳

### Q3: 数据积累多久可以启用 Phase 2？

**A**: 
- **最少**: 5 天（静默积累）
- **推荐**: 30 天（充足样本）
- **最佳**: 60 天（高置信度）

### Q4: Phase 4 的建议准确率能达到多少？

**A**: 
- **理论目标**: 胜率 > 55%（跑赢随机）
- **实际预期**: 高置信建议（>70%）胜率 > 65%
- **不保证收益**：历史规律不代表未来

### Q5: 是否需要投顾牌照？

**A**: 
- **法律灰色地带**：建议咨询律师
- **风险控制**：
  - 明确免责声明
  - 不承诺收益
  - 强调"辅助工具"而非"投资建议"
  - 不收取投顾费用

### Q6: 如何避免误导用户？

**A**: 
- ✅ 展示概率，不承诺收益
- ✅ 明确风险提示（最大回撤、失败概率）
- ✅ 强调数据局限性（样本数、时间范围）
- ✅ 免责声明置顶
- ❌ 避免"保证"、"必然"等绝对表述

### Q7: 系统能否完全自动化交易？

**A**: 
- **设计定位**: 辅助决策，不自动交易
